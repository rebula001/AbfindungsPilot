## Check-Skript

[`scripts/check-i18n-keys.mjs`](../../../frontend/scripts/check-i18n-keys.mjs) prÃžft drei Dinge:

| Befund                | Bedeutung                                                       | Aktion                                                                                     |
| --------------------- | --------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| **Missing**           | Source-Code referenziert Key, der in `de.ts` oder `zh.ts` fehlt | hinzufÃžgen oder Tippfehler korrigieren                                                     |
| **Cross-locale diff** | Key in DE vorhanden, in ZH fehlt (oder umgekehrt)               | nach-Ãžbersetzen                                                                            |

Erkennungsstrategie (vereinfacht):

- `` `a.b.c` `` / `"a.b.c"` / `'a.b.c'` direkt -> verwendet

- `"a.b.${x}.c"` -> Keys mit PrÃĪfix `a.b.` und Suffix `.c` markiert
- `t('a.b')` -> `a.b` und alle Children markiert
- `t('a.b.c.d')` -> alle Eltern (`a.b`, `a.b.c`) auch markiert

**Exit-Code:** Missing oder Cross-locale-Diff > 0 -> `1` (lÃĪsst `npm run build` fehlschlagen). Unused -> nur Report, kein Fail.
