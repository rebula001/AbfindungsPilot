# 05 - i18n

> **30 秒了解：** `vue-i18n` 11，两种 locale（DE 默认、ZH）。`de.ts` 和 `zh.ts` **结构 1:1** 必须一致（vue-i18n 通过 TypeScript 强制）。提交前：`npm run i18n:check`。

## 配置

[`src/i18n/index.ts`](../../../frontend/src/i18n/index.ts)：

```ts
export const i18n = createI18n({
  legacy: false,
  locale: loadInitialLanguage(), // localStorage `abfindungspilot.language.v1`，否则 'de'
  fallbackLocale: 'de', // 未翻译的 ZH key -> 显示 DE 内容
  warnHtmlMessage: false, // Tooltip 含 <p>/<ul>/<strong>
  messages: { de, zh }
});
```

`fallbackLocale: 'de'` 的意思：缺失的 ZH key 会显示德语，而不是冒出 `t('xxx')` 这样的占位符。**但这仍然是 bug**，check 脚本会报。

## 结构规则 - DE 和 ZH 同构

由于 vue-i18n 的 TypeScript 类型推导，`vue-tsc -b`（在 `npm run build` 中）强制 `de` 和 `zh` 拥有完全一致的 key 树。结构差异 -> build 失败。

```ts
// ❌ 会让 build 挂掉：
// de.ts: { foo: { bar: 'X' } }
// zh.ts: { foo: { baz: 'Y' } }    // bar 缺失、baz 多出来

// ✅ 正确：
// de.ts: { foo: { bar: 'X' } }
// zh.ts: { foo: { bar: 'Y' } }
```

## 翻译规则 - 保留德语税务术语

中文翻译里 **不要把德语税务术语翻死**，因为：

1. 它们是没有精确中文对应的专有概念
2. 用户在 Steuerbescheid（税单）和咨询场景里反而会再看到原文
3. App 的目标用户是在德国的中文使用者

| DE | ZH |
| --- | --- |
| `Abfindung` | `Abfindung`（遣散费）（首次出现），之后 `Abfindung` |
| `KFB` | `KFB`（儿童免税额）（1x），之后 `KFB` |
| `Soli` | `Soli`（团结附加税）（1x），之后 `Soli` |
| `Fünftelregelung` | `Fünftelregelung`（五分之一规则）（1x），之后 `Fünftelregelung` |
| `Progressionsvorbehalt` | `Progressionsvorbehalt`（累进保留）（1x），之后 `Progressionsvorbehalt` |
| `ALG I` | `ALG I`（失业救济金）（1x） |
| `GKV` / `PKV` | `GKV`（法定医保） / `PKV`（私人医保） |
| `zvE` | `zvE`（应税所得） |
| `Veranlagungsart` | `Veranlagungsart`（申报方式） |
| `Splittingtarif` | `Splittingtarif`（分摊税率） |
| `§ 32a EStG`、`§ 158 SGB III` | 原样保留 |

联邦州名保留德语（`Bayern`、`Baden-Württemberg`）。

## 多行内容：`tList()` 与 `tm()`

需要列举的内容（解释段、公式步骤）在 locale 文件里存数组，模板层用 `tm()` 迭代。参考 `CalculationView.vue`。

```ts
// de.ts
calculation: {
  fuenftelExplanation: ['Schritt 1: ...', 'Schritt 2: ...', 'Schritt 3: ...']
}
```

```vue
<li v-for="(line, i) in tm('calculation.fuenftelExplanation')" :key="i">{{ line }}</li>
```

好处：每行可单独翻译，比一大段 HTML 字符串好维护。

## HTML Tooltip

部分 tooltip 需要排版（`<p>`、`<ul>`、`<strong>`）。允许，因为：

- 内容只来自静态 locale 文件（无用户输入 -> 无 XSS 风险）
- `warnHtmlMessage: false` 关掉了 intlify 的警告

通过 `v-tooltip` + `escape: false` 接入：

```vue
<button
  type="button"
  class="pi pi-info-circle ..."
  v-tooltip.right="{
    value: t('form.taxClassTooltip'),
    escape: false,
    class: 'tax-class-tooltip',
    showDelay: 150
  }"
  :aria-label="t('form.taxClassTooltipAria')"
/>
```

## 语言切换

Toolbar 右上角两个按钮 DE / ZH。点击 -> `cacheLanguage(lang)` 写 localStorage 并设 `i18n.locale.value`。

持久化 key：`abfindungspilot.language.v1`（**无** TTL，与 snapshot key 不同）。

## Check 脚本

[`scripts/check-i18n-keys.mjs`](../../../frontend/scripts/check-i18n-keys.mjs) 检查三件事：

| 报告 | 含义 | 处理 |
| --- | --- | --- |
| **Missing** | 源代码引用了 key，但 `de.ts` 或 `zh.ts` 里没有 | 补上 / 改拼写 |
| **Cross-locale diff** | DE 里有，ZH 里缺（或反之） | 补翻译 |
| **Unused** | locale 里有，源代码从未用到 | 候选删除 - 人工确认（动态拼接的 key 静态分析不到） |

识别策略（简化）：

- `` `"a.b.c"` / `"a.b.c"` / `'a.b.c'` `` 直接 -> 视为已用
- `` "a.b." + var `` -> 标记 `a.b.*`
- `` `a.b.${x}.c` `` -> 标记前缀 `a.b.`、后缀 `.c` 的所有 key
- `t('a.b')` -> `a.b` 及其所有子节点视为已用
- `t('a.b.c.d')` -> 所有上级（`a.b`、`a.b.c`）也视为已用

**退出码：** Missing 或 Cross-locale-Diff > 0 -> `1`（CI 阻塞）。Unused -> 仅报告，不阻塞。

## 加新 UI 文本的流程

1. 在 `de.ts` 合适位置加 key
2. 在 `zh.ts` 同位置加翻译（保留德语税务术语！）
3. 模板 / TS 中用 `t('xxx')` 引用
4. `npm run i18n:check` -> 必须通过
5. `npm run build` -> `vue-tsc` 确认结构一致
